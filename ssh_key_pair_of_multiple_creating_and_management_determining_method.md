# 【備忘録】SSHキーペアの複数作成と管理・判別方法

本ドキュメントは、1つのデバイス（PC）内に用途別のSSHキーペア（Ed25519等）を複数作成する方法と、`.ssh` ディレクトリ内で秘密鍵と公開鍵のペアが分からなくなった際の判別方法をまとめたものである。

---

## 1. 1つのデバイスで複数のキーペアを作成する方法

SSHのキーペアは、1つのデバイス内に**無制限**に作成・共存させることができる。
デフォルトのまま作成すると `id_ed25519` という固定名になり上書きされてしまうため、作成時に **`-f` オプション**を使用してファイル名を明示的に指定する。

### コマンド例（Ed25519規格の場合）

```bash
# AWS用のキーペアを作成
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519_aws

# GitHub個人アカウント用のキーペアを作成
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519_github
```

※パスフレーズの入力を求められた場合は、必要に応じて入力（またはEnterで省略）する。

この操作により、指定した名前で「秘密鍵」と「公開鍵（`.pub`）」のペアがそれぞれ生成される。

---

## 2. 秘密鍵と公開鍵の「正しいペア」を判別する方法

複数のキーペアを作成し、どれとどれがペアなのか分からなくなった場合、以下の2つのアプローチで確実に判別できる。

### 方法A：ファイル名（拡張子）で判断する（基本）

SSHの仕組み上、キーペアは必ず**「拡張子なし（秘密鍵）」**と**「`.pub`（公開鍵）」**の同じファイル名で生成される。ファイル名を変更していない限り、この法則でペアを特定できる。

- **ペア1:** `id_ed25519_aws`（秘密） ＆ `id_ed25519_aws.pub`（公開）
- **ペア2:** `id_ed25519_github`（秘密） ＆ `id_ed25519_github.pub`（公開）

### 方法B：秘密鍵から公開鍵を計算して割り出す（確実な方法）

ファイル名を変更してしまった場合や、本当に正しいペアか確認したい場合に用いる手法。
公開鍵暗号方式の特性上、**「秘密鍵」さえあれば、そこから対になる「公開鍵」を100%復元（再生成）することが可能**である。（※公開鍵から秘密鍵を復元することは絶対にできない）

以下のコマンドを実行することで、指定した秘密鍵から導き出された正しい公開鍵の文字列が画面に出力される。

```bash
# 秘密鍵から公開鍵の文字列を出力するコマンド
ssh-keygen -y -f ~/.ssh/id_ed25519_aws
```

**【確認手順】**

1. 上記のコマンドを実行し、画面に出力された文字列（`ssh-ed25519 AAA...`）をコピーする。
2. 手元にある疑わしい公開鍵ファイル（例: `id_ed25519_aws.pub`）をテキストエディタや `cat` コマンドで開く。
3. 双方の文字列が**完全に一致**していれば、その2つのファイルは間違いなくペアであると断言できる。
