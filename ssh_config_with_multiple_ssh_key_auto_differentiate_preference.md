# 【備忘録】 ~/.ssh/config を用いた複数SSHキーの自動使い分け設定

本ドキュメントは、1つのデバイス内に複数作成したSSHキーペア（AWS用、GitHub用など）を、接続先に応じて自動的に使い分けるための `~/.ssh/config` の実践的な設定方法をまとめたものである。

---

## 1. なぜ `config` ファイルが必要なのか？

複数のSSHキーを作成した場合、単に `ssh <ユーザー名>@<IPアドレス>` とコマンドを打つだけでは、以下の2つの問題が発生する。

1. **デフォルトキーの限界:** SSHはデフォルトで `~/.ssh/id_ed25519` や `~/.ssh/id_rsa` などの決まった名前の鍵しか探しに行かない。自分で名前を付けた鍵（例: `id_ed25519_aws`）は、明示的に指定しない限り無視されてしまう。
2. **認証エラー（Too many authentication failures）:** 鍵の指定を省略した場合、SSHは「とりあえず持っている鍵を片っ端から試す」という挙動をすることがある。一定回数間違った鍵でアタックすると、サーバー側から「不正アクセスの疑いあり」として接続を強制切断されてしまう。

これを解決するのが `~/.ssh/config` ファイルである。このファイルに「この宛先には、このユーザー名で、この鍵を使って接続する」というルール（ルーティング表）を書いておくことで、**鍵の指定漏れやアタック判定を完全に防ぎ、さらに接続コマンドを極限まで短縮できる。**

---

## 2. `config` ファイルの作成と基本構文

### 2-1. ファイルの作成

ご自身のPCの `.ssh` ディレクトリ内に、拡張子なしで `config` という名前のテキストファイルを作成する。

- **Windows:** `C:\Users\<PCのユーザー名>\.ssh\config`
- **Mac/Linux:** `~/.ssh/config`

### 2-2. 基本構文

ファイルの中身は、以下の4つの項目を1セットとして記述していく。（インデントはスペースでもTabでも可）

```text
Host <任意のショートカット名>
    HostName <実際の接続先ドメイン名 または IPアドレス>
    User <ログインするユーザー名>
    IdentityFile <使用する秘密鍵の絶対パス または ~/.ssh/からのパス>
```

---

## 3. 実践的な設定例（AWS と GitHub の使い分け）

開発現場で最もよく使われる「AWS（EC2）」と「GitHub（個人用・会社用）」を自動で振り分ける設定例。

```text
# ==========================================
# 1. AWS EC2 インスタンス用
# コマンド: ssh aws-dev
# ==========================================
Host aws-dev
    HostName <任意のサブドメイン>.duckdns.org
    User ubuntu
    IdentityFile ~/.ssh/id_ed25519_aws

# ==========================================
# 2. GitHub (メインの個人アカウント用)
# コマンド: ssh -T git@github.com
# ==========================================
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_github_personal

# ==========================================
# 3. GitHub (サブのチーム開発/会社アカウント用)
# コマンド: ssh -T git@github-work
# ==========================================
Host github-work
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_github_work
```

---

## 4. 【超重要テクニック】GitHubの複数アカウント使い分け手法

上記の設定例「3. GitHub（サブ）」には、エンジニア必須のテクニックが隠されている。
GitHubはすべての接続で `git` という同一ユーザー名を使用し、ドメインも同じ `github.com` であるため、通常はどのアカウントで接続したいのかSSH側が判別できない。

そこで、**`Host` に架空のショートカット名（例: `github-work`）を設定**する。

### 使い方（Gitクローン時の変更）

会社用（サブ）のアカウントのリポジトリを `git clone` する際、通常のURLではなく、`config` で設定した架空のショートカット名に書き換えて実行する。

- **通常のクローンコマンド（エラーになるか、メインアカウントで認証される）:**
  ```bash
  git clone git@github.com:TeamName/Project.git
  ```
- **架空のショートカット名を使ったクローンコマンド（正しくサブアカウントの鍵が使われる）:**
  ```bash
  git clone git@github-work:TeamName/Project.git
  #              ^^^^^^^^^^^
  # configのHost名と一致させることで、自動的に id_ed25519_github_work の鍵が選ばれる
  ```

この設定をしておけば、プロジェクトごとに異なるSSHキー（GitHubアカウント）をシームレスに使い分けることができる。
