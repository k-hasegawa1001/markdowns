# Docker Hub ログイン方法メモ

## 前提環境

- **環境**: Windows (WSL2)
- **アカウント**: GitHub連携でサインアップ済み（メールアドレス・パスワードでサインアップした場合は[方法3](#方法3メールアドレスパスワードで登録した場合通常アカウント)を参照すること）

---

## 方法1：Webベース認証（今回成功した方法）

Docker Desktopなどが導入されている場合や、最新のCLIで利用できる最も簡単な方法。

1. WSLのターミナルでコマンドを実行。

   ```bash
   docker login
   ```

2. ターミナルに以下のようなメッセージが表示される。

   Please visit https://login.docker.com/activate and enter code XXXX-XXXX

3. Windows側のブラウザで https://login.docker.com/activate を開く。

4. ターミナルに表示された 8桁のコード をブラウザに入力して承認（Confirm）する。

5. ターミナルに戻ると自動的に `Login Succeeded` となり完了。

## 方法2：アクセストークン認証（汎用的な方法）

Web認証が画面に出ない場合や、SSH等のCUI環境、CI/CDパイプラインで使用する方法。

**1. トークンの発行**

1.  ブラウザで [Docker Hub](https://hub.docker.com/) にログインする。

2.  右上のアイコン > **Account Settings** > **Security** を開く。

3.  **[New Access Token]** をクリック。

4.  権限（Access permissions）を `Read, Write, Delete` に設定し、説明（Description）を入力して生成。

5.  **表示されたトークン文字列をコピーする**（この画面を閉じると二度と表示されないため注意）。

**2. コマンドでのログイン**

1.  WSLのターミナルでコマンドを実行。

    ```Bash
    docker login
    ```

2.  **Username**: Docker ID を入力。
    - ※ メールアドレスではないので注意。

    - ※ 右上アイコンの横に表示されている名前。

3.  **Password**: 先ほどコピーした アクセストークン を貼り付け。
    - ※ GitHubのパスワードではない。

    - ※ Linuxの仕様上、貼り付けても画面には文字も記号も表示されないが、そのままEnterを押す。

# 注意点

- **GitHubパスワードは使えない**: GitHub連携アカウントの場合、docker loginのパスワード欄にGitHubのパスワードを入れても認証エラーになる。必ずアクセストークンを使用する。

- **ユーザー名**: 基本的にメールアドレスではなく「Docker ID」を使用する。

## 方法3：メールアドレス・パスワードで登録した場合（通常アカウント）

GitHub連携などを利用せず、Docker Hubで直接アカウントを作成した場合の標準的な方法。

### 手順

1. WSLのターミナルでコマンドを実行。

   ```bash
   docker login
   ```

2. **Username**: `Docker ID` を入力。
   - ※ メールアドレスを入力すると失敗する場合があるため、Docker ID（ユーザー名）推奨。

3. **Password**: 登録時に設定した `ログインパスワード` を入力。
   - ※ 入力中は画面に文字も記号も表示されないが、そのままEnterを押す。

**⚠️ 注意：2段階認証（2FA）を設定している場合**\
セキュリティ設定で2FAを有効にしている場合は、**通常のパスワードではログインできません**（エラーになります）。\
その場合は「[方法2](#方法2アクセストークン認証汎用的な方法)」と同様に、Docker Hub上で アクセストークン（**Personal Access Token**） を発行し、パスワードの代わりにそれを入力してください。
