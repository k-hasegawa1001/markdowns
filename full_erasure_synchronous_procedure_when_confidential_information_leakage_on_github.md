# GitHubでの機密情報（.env等）漏洩時の完全抹消・同期手順

## 🚨 フェーズ0：最優先事項（絶対に最初に行うこと）

GitHubのパブリック領域で一度でも公開された機密情報は、Gitの履歴から消し去っても「既に悪意ある第三者に取得された（漏洩済み）」とみなすのが鉄則です。
Gitの復旧操作を行う前に、**必ず該当サービス（AWS, OpenAI等）の管理画面から対象のトークンやキーを「無効化（Revoke）」し、再発行**してください。

---

## 🛠️ フェーズ1：歴史の書き換えとリモートへの強制反映（実行者：1名）

機密情報を含んだ過去のコミット履歴を再構築し、ファイルそのものの存在を歴史から完全に抹消します。

### 1. `git filter-repo` のインストール

Python環境が前提となります。以下のコマンドで外部ツールをインストールします。

```bash
pip install git-filter-repo
```

### 2. 新規クローン（安全のための隔離環境）

作業ミスによる二次被害を防ぐため、現在作業中のディレクトリは使わず、必ず新しくクローンしたまっさらなディレクトリで作業します。

```bash
git clone [https://github.com/ユーザー名/リポジトリ名.git](https://github.com/ユーザー名/リポジトリ名.git)
cd リポジトリ名
```

### 3. 歴史の書き換え（特定ファイルの抹消）

プロジェクトの誕生から現在までの全歴史から `.env` を削除し、全く新しいコミットハッシュ値で歴史を再構築します。

```bash
git filter-repo --invert-paths --path .env
```

### 4. リモートリポジトリへの強制プッシュ

新しく作られた「クリーンな歴史」で、GitHub上の古い歴史を強制的に上書きします。（`filter-repo` 実行後はリモート設定が外れるため、再登録が必要です）

```bash
git remote add origin [https://github.com/ユーザー名/リポジトリ名.git](https://github.com/ユーザー名/リポジトリ名.git)
git push origin --force --all
```

---

## ⚠️ フェーズ2：チームメンバーの環境同期（実行者：他の全開発メンバー）

強制プッシュ後、他のメンバーがローカル環境で通常の同期操作を行うと**機密情報が復活する大事故**に繋がります。必ず以下の手順を周知・徹底してください。

### 【厳禁】絶対にやってはいけないコマンド

❌ `git pull` または `git merge`
（手元の「機密情報入りの古い歴史」と「新しい歴史」がマージされ、機密情報が復活した状態で再度プッシュされてしまいます）

### 同期手順（推奨：再クローン方式）

最も安全で確実な方法です。手元に未プッシュの作業データがある場合は、**実行前に必ず該当ファイルだけをリポジトリのフォルダ外へ退避（バックアップ）**させてください。

```bash
# 1. 古いローカルリポジトリを削除（または別名にして退避）
rm -rf リポジトリ名

# 2. 新しい歴史になったリポジトリを再度クローン
git clone [https://github.com/ユーザー名/リポジトリ名.git](https://github.com/ユーザー名/リポジトリ名.git)
```

#### ※（代替案）コマンドで強制同期する場合

既存のディレクトリをそのまま生かし、リモートの最新状態に強制上書きします。（未コミットの変更はすべて消去されます）

```bash
git fetch origin
git reset --hard origin/main  # masterブランチの場合は origin/master
```
