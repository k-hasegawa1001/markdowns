# ⚓ Elastic IP (固定IP) の設定手順

このドキュメントでは、サーバーを再起動してもIPアドレスが変わらないようにする「Elastic IP」の設定方法を解説します。

**なぜこれが必要？**
AWSのEC2は、通常の設定だと**再起動するたびにIPアドレスが変わってしまいます**。
IPが変わると以下の問題が起きます。

- ターミナルからSSH接続できなくなる（設定し直し）
- 自動デプロイ (CI/CD) が失敗する
- Webサイトが表示されなくなる

---

## 1. 【重要】AWS Academyでのコスト注意点

AWS Academy (Learner Lab) 環境では、Elastic IPについて厳しいルールがあります。

> **Elastic IPを取得している間は、EC2を停止していても「$0.005 / 時間」の予算が消費され続けます。**

1ヶ月放置すると約 $3.6 (約500円) 消費します。
授業期間外など、数週間以上サーバーを使わない場合は、必ずElastic IPを**「解放 (Release)」**してください。

---

## 2. 【手順】IPアドレスの取得と紐づけ

AWSコンソールの画面は少しわかりにくいので、以下の手順通りに進めてください。

### STEP 1: IPアドレスの確保 (Allocate)

まず、AWSからIPアドレスを1つ借り受けます。

1.  AWSコンソールの左メニューから **「ネットワークとセキュリティ」 > 「Elastic IP」** をクリック。
2.  右上のオレンジ色のボタン **「Elastic IP アドレスを割り当てる」** をクリック。
3.  設定項目は変更せず、そのまま右下の **「割り当て」** をクリック。
4.  緑色の帯で「Elastic IP アドレスが正常に割り当てられました」と表示されればOKです。

### STEP 2: サーバーへの紐づけ (Associate)

**⚠️ ここが最重要です！まだ作業は終わっていません！**
確保したIPを、あなたのEC2インスタンスにくっつける作業が必要です。

1.  今作成されたIPアドレス（`54.x.x.x` など）の左側にあるチェックボックスにチェックを入れる。
2.  右上の **「アクション」** プルダウンメニューから **「Elastic IP アドレスの関連付け」** を選択。
3.  設定画面で以下を入力する：
    - **リソースタイプ:** 「インスタンス」を選択。
    - **インスタンス:** 入力欄をクリックし、自分のサーバー（Running中のもの）を選択。
    - **プライベートIPアドレス:** 入力欄をクリックし、出てきたIPアドレス（`172.x.x.x`）を選択。
4.  右下の **「関連付け」** をクリック。

---

## 3. 確認

EC2のダッシュボード（インスタンス一覧）に戻り、自分のインスタンスを選択してください。
「パブリック IPv4 アドレス」の欄に、先ほど取得したElastic IPが表示されていれば成功です！

これ以降、SSH接続やブラウザでのアクセスは、すべてこの **新しいIPアドレス** を使用してください。
